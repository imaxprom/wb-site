# E2E Чеклист — Расчёт отгрузки (/shipment)

## Бизнес-инварианты (ВСЕГДА должны быть истиной)

### Числовые
- [ ] Все числа в таблицах >= 0 (нет NaN, Infinity, отрицательных)
- [ ] «Нужно» (V1) >= 0 для каждого размера/региона
- [ ] «Кор.» (V2) >= 0 для каждого размера/региона
- [ ] Smart коробов >= 0
- [ ] На складе WB > 0 (если данные загружены)
- [ ] Потребность >= 0
- [ ] Множитель тренда в диапазоне 0.1 – 3.0

### Динамика по неделям
- [ ] Количество недель = wb-upload-days / 7
- [ ] Все недели имеют диапазон дат (формат DD.MM – DD.MM)
- [ ] Даты идут последовательно (Нед.1 < Нед.2 < Нед.3 < Нед.4)
- [ ] Сегодняшний (неполный) день НЕ входит в последнюю неделю
- [ ] Сумма заказов по неделям ≈ общее количество заказов (допуск: заказы сегодня)
- [ ] R² в диапазоне 0.00 – 1.00
- [ ] Прогнозная строка присутствует и >= 0

### Регионы
- [ ] Сумма % по всем регионам = 100% (±1% на округление)
- [ ] Каждый регион имеет shortName
- [ ] Факт >= 0 для каждого региона

### Выкуп
- [ ] Авто-режим: показывает % из realization (delivery_amount/quantity), ~80-83%
- [ ] Авто-режим: таблица "Реальный % выкупа" показывает Выкупы/Отказы (не Заказы/Отмены)
- [ ] Ручной режим: показывает точный % (NN% ручной)
- [ ] % выкупа в диапазоне 10-100%
- [ ] Источник: `/api/data/buyout-rates` (realization с дедупликацией weekly_final > weekly > daily)

### V3 Smart Packing
- [ ] Если остатки = 0 по размеру → он ОБЯЗАТЕЛЬНО грузится (не фильтруется)
- [ ] Дефицит < minUnits → позиция в секции «Отложено»
- [ ] Заполненность короба 0-100%
- [ ] Количество позиций в коробе <= maxArticlesPerBox

## UI-проверки

### Вкладки
- [ ] 4 вкладки видны: Расчёт, Товары, Загрузка данных, Настройки отгрузки
- [ ] Переключение между вкладками работает
- [ ] Активная вкладка визуально выделена

### Режимы V1/V2/V3
- [ ] V1: без столбцов «Шт/кор» и «Коробки»
- [ ] V2: со столбцами «Шт/кор», «V1», «V2 тренд», «Коробки»
- [ ] V2: панель динамики по неделям видна
- [ ] V3: 4 карточки (Smart/Склад/Потребность/Баланс) видны
- [ ] V3: настройки справа от графика (Позиций/кор, Отгрузок/мес, Мин.штук, Вид)
- [ ] Переключатель V1/V2/V3 работает

### Загрузка данных
- [ ] Диапазон дат отображается рядом с селектом (DD.MM – DD.MM)
- [ ] Селект: 4/5/6/7/8 недель
- [ ] Кнопка «Загрузить всё из WB» видна
- [ ] Секция «Текущие данные» показывает заказы/остатки/артикулы

### Настройки отгрузки
- [ ] % выкупа: кнопки Авто/Ручной
- [ ] Авто → шкала % скрыта
- [ ] Ручной → шкала % появляется
- [ ] Регионы: кнопки Авто/Ручной
- [ ] Размер короба: 3 поля (Длина/Ширина/Высота)
- [ ] Кнопка «Сохранить»

### Фильтр артикулов
- [ ] «Все артикулы» по умолчанию
- [ ] При выборе конкретного артикула — таблица фильтруется
- [ ] Чекбокс «Скрыть неактивные» работает

## Верификация данных (ОБЯЗАТЕЛЬНО после каждой загрузки)

### Остатки
- [ ] Total stock в UI = SUM(quantity) из SQLite
- [ ] Total stock в UI = total из WB API supplier/stocks (dateFrom=2019)
- [ ] Выборочно: 2-3 баркода — количество совпадает с ЛК WB
- [ ] Количество складов на баркод — не подозрительно мало (должно быть 10-60+)

### Заказы
- [ ] Заказы в UI = count из SQLite за выбранный период
- [ ] Сумма по неделям = заказы в UI
- [ ] Нет заказов за сегодня (неполный день)

### Регионы
- [ ] Итого = 100%
- [ ] Количество заказов в основе = количество заказов в UI

### Потребность
- [ ] Потребность = сумма дефицитов (план − факт) по позициям с дефицитом
- [ ] Потребность ≤ прогноз заказов (не может быть больше)
- [ ] Smart коробов × perBox ≈ потребность (с учётом отложенных)

## Мутационные проверки (изменение → ожидание)

| Действие | Что должно измениться |
|----------|----------------------|
| Сменить V1→V2 | Появляются столбцы динамики + график |
| Сменить V2→V3 | Появляются карточки Smart + укладка |
| Сменить выкуп Авто→Ручной | Появляется шкала % |
| Сменить период 4→5 недель | Диапазон дат расширяется |
| Выбрать 1 артикул | Строк становится меньше |
| Убрать «Скрыть неактивные» | Строк может стать больше |
