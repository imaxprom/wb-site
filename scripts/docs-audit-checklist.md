## КРИТИЧЕСКИЕ ПРАВИЛА

1. **ЗАПРЕЩЕНО использовать память, кэш, предыдущий контекст.** Каждый файл открывай и читай заново через Read/cat. Даже если ты "помнишь" содержимое — перечитай.
2. **Источник истины — только код и БД.** Не docs.json, не CLAUDE.md, не твои знания. Код = правда. Документация = то, что нужно привести в соответствие с кодом.
3. **Один раздел за раз.** Не переходи к следующему разделу, пока не закончил текущий. Фокус на одном.
4. **После каждого изменения — проверка.** `npx tsc --noEmit` + `./scripts/test-api.sh test`. Если тесты упали — откати и разберись.
5. **Не меняй код приложения.** Только docs.json, REFACTORING.md, mphub-database-guide.md, e2e-checklist.md. Если нашёл баг в коде — запиши в отчёт, но не исправляй.

## ШАГ 2: Сверка с docs.json

### 2.1 Прочитай текущее описание раздела в docs.json
```bash
python3 -c "
import json
with open('public/data/docs.json') as f:
    d = json.load(f)
for s in d['sections']:
    if s['id'] == 'РАЗДЕЛ':
        print(json.dumps(s, ensure_ascii=False, indent=2))
"
```

### 2.2 Сверь КАЖДЫЙ блок

Для каждого блока в разделе:

**Тип `text` (описание):**
- [ ] Описание соответствует тому, что ты увидел в коде?
- [ ] Источник данных указан правильно?
- [ ] Упомянуты ли Error Boundary, провайдеры, модульная структура?

**Тип `list` (списки элементов):**
- [ ] Каждый элемент списка существует в реальности?
- [ ] Нет ли элементов, которых уже нет? (удалённые табы, переименованные компоненты)
- [ ] Нет ли пропущенных элементов? (новые API, новые файлы, новые карточки)
- [ ] Пути файлов корректны? (src/components/ vs src/modules/РАЗДЕЛ/components/)

**Тип `schema` (потоки данных):**
- [ ] Поток данных соответствует реальному коду?
- [ ] API-эндпоинты в потоке существуют?
- [ ] Количество элементов на выходе правильное? (4 карточки vs 5)

**Тип `code` (исходный код):**
- [ ] Описание в title актуально?
- [ ] Путь файла правильный?

### 2.3 Проверь пути файлов
Для КАЖДОГО файла, упомянутого в docs.json:
```bash
test -f "ПУТЬ" && echo "✅ EXISTS" || echo "❌ MISSING: ПУТЬ"
```
Если файл не существует — удали из docs или исправь путь.

### 2.4 Проверь API-эндпоинты
Для каждого API, упомянутого в docs.json:
```bash
test -f "src/app/api/ПУТЬ/route.ts" && echo "✅" || echo "❌ MISSING"
```
