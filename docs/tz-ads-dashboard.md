# ТЗ: Дашборд управления рекламой WB

> Версия 2.0 — полная ревизия. Все API routes проверены по исходному коду проекта.

---

## 1. Общее описание

Новая страница `/ads` — дашборд для управления рекламными кампаниями Wildberries. Главный элемент — горизонтально-скроллируемая таблица товаров с рекламными метриками за выбранный период.

**Стек (СТРОГО из проекта):**
- Next.js App Router + TypeScript
- Tailwind CSS 4
- Без дополнительных UI-библиотек (кроме уже установленных в проекте)

**Цветовая гамма:** тёмная тема проекта. Основные цвета брать из существующих страниц (`src/app/finance/page.tsx`, `src/app/shipment/page.tsx`). Фиолетово-пурпурные акценты.

### 2.2. Панель фильтров

Одна горизонтальная строка под табами, элементы слева направо:

#### 2.2.1. Кнопка обновления
- Иконка обновления (SVG, не эмодзи)
- При клике: вызывает `POST /api/wb/adv` для принудительного sync рекламных данных с WB API
- Во время загрузки: иконка крутится (animation: spin)

#### 2.2.2. Поле поиска
- Текстовый input, placeholder: "Артикул"
- Фильтрация по: nmId (число) или vendorCode (строка артикула продавца)
- Рядом — текст-счётчик: `N (TOTAL)` — сколько строк показано из общего количества
- Фильтрация на клиенте (данные уже загружены)

#### 2.2.3. Режим группировки
- Выпадающий список (select)
- Варианты: `по склейке` | `по баркоду`
- По умолчанию: `по склейке`
- **Заглушка:** логика группировки — на первом этапе просто показывать каждый nmId как отдельную строку. Группировку по склейке реализовать позже.

#### 2.2.4. Чекбокс "Архив"
- Отображение: `☐ Архив`
- Если включён — показывать также товары без остатков и без заказов за период
- По умолчанию: выключен (архивные скрыты)

#### 2.2.5. Период
- Выпадающий список (select)
- Варианты (СТРОГО по скриншоту):

| Неделя | 7 |
| 10 дней | 10 |

По умолчанию: **Сегодня**

#### 2.2.6. Сводка за период
- Текст справа: `За сегодня* XXX XXX Р (-XX XXX Р)`
- Первое число — сумма заказов за период
- Второе число в скобках — изменение затрат на рекламу за период
- **Заглушка:** на первом этапе можно показывать только сумму заказов или пустую строку

## 3. Основная таблица

### 3.0. Общее поведение таблицы

- Горизонтальный скролл для столбцов правее "Товар"
- Столбец "Товар" **зафиксирован слева** (CSS `sticky`)
- При наведении на строку — подсветка всей строки (hover эффект)
- Все числовые значения форматировать с разделителями тысяч (пробелы): `3 858 000`, не `3858000`
- Денежные значения с суффиксом `Р` (без пробела перед Р): `38 318 Р` — хотя, можно `38 318 ₽`

### 3.1. Сортировка (общие правила)
- Клик по заголовку столбца переключает сортировку: **нет → по убыванию → по возрастанию → нет**
- Стрелка ↓/↑ рядом с названием столбца показывает активную сортировку
- Одновременно активна **только одна** сортировка
- В каждом столбце ниже указано, по какому полю идёт сортировка

### 3.2. Тултипы (всплывающие подсказки)
- При наведении на **заголовок** столбца — показывать tooltip с описанием (текст из секций "Тултип" ниже)
- Тултип — абсолютно позиционированный div под курсором, тёмный фон, белый текст, скруглённые углы
- Появляется с задержкой 300ms, исчезает при уходе курсора

    ```sql
    SELECT product_article,
    ```

### Столбец 9: ДРРз

**Ширина:** ~60px.

**Содержимое:** процент (одно число).

**Формула:**
```
ДРРз = (затраты_на_рекламу / сумма_всех_заказов) × 100
```

**Особые случаи отображения:**
| Ситуация | Отображение |
|---|---|
| ДРРз >= 1% | Число с одним десятичным: `8.3 %` |
| ДРРз > 0 но < 1% | `0.X %` (показывать десятичные, ведущий 0) |
| ДРРз > 0 но < 0.1% | `~ %` (тильда — значение очень маленькое) |
| Есть расход, но нет заказов | `# %` (решётка — нет заказов) |
| Нет расхода и нет заказов | `—` (прочерк) |

**Тултип заголовка:**
```
Доля рекламных расходов по отношению к сумме всех заказов
для значений меньше 1% ведущий 0 'целых' не показывается
# % — нет заказов, хотя есть расход по рекламе
~ — сильно меньше 0,1 %
```

**Расчёт:** полностью на клиенте из данных столбцов "Затраты рекл." и "Заказы общ."

**Сортировка:** нет (на скриншотах не указана).

### Столбец 10: Затраты на рекламу (Затраты рекл.)

**Ширина:** ~150px.

**Содержимое (3 строки):**
1. 🛒 `643` × `27 Р` — корзины с рекламы × рекламная себестоимость корзины
2. 🛒 `148` × `116 Р` — заказы с рекламы × рекламная себестоимость заказа
3. `17 113 Р` — общие затраты на рекламу (сумма начислений)

**Формулы:**
```

```

**Пометка `+?`:** появляется если у товара есть корзины/заказы, пришедшие от рекламы ДРУГИХ товаров (т.е. пользователь кликнул рекламу товара A, но купил товар B).

### 4.2. Новые API routes (нужно создать)

#### `GET /api/ads/dashboard`

**Файл:** `src/app/api/ads/dashboard/route.ts`

Главный агрегирующий endpoint. Собирает данные из нескольких источников и возвращает единый ответ для таблицы.

**Параметры:**
- `period` — количество дней (1, 2, 3, 5, 7, 10, 14, 30, 60, 90). По умолчанию: 1
- `search` — фильтр по nmId или vendorCode (опционально)

**Логика:**
1. Параллельно запросить:
   - Карточки товаров (WB Content API через `/api/wb/cards`)
   - Остатки (WB Statistics API через `/api/wb/stocks`)
   - Заказы (WB Statistics API через `/api/wb/orders?days={period}`)
   - Рекламные расходы по nmId (SQL из DB `advertising`)
   - Кампании (WB Advert API — список + маппинг + расходы)
2. Объединить по nmId
3. Рассчитать: стоимость остатков, сумму заказов, ДРРз
4. Отфильтровать по search, archive

**Типы ответа:**

```typescript
// === src/types/ads.ts ===

/** Ответ главного endpoint дашборда */
interface AdsDashboardResponse {
  summary: {
    totalOrdersSum: number;     // сумма заказов за период (все товары)
    totalAdsSpend: number;      // сумма затрат на рекламу (все товары)
    totalProducts: number;      // всего товаров
    filteredProducts: number;   // показано после фильтрации
  };
  products: AdsDashboardProduct[];
}

/** Одна строка таблицы — один товар */
interface AdsDashboardProduct {
  // === Столбец "Товар" ===
  nmId: number;                  // код товара WB
  vendorCode: string;            // артикул продавца
  title: string;                 // название товара
  updatedAt: string;             // время последнего редактирования карточки (ISO)

  // === Столбец "Предмет" ===
  subject: string;               // категория ("Трусы", "Носки")

  // === Столбец "Цвета" ===
  colors: string[];              // ["чёрный", "белый", "серый"]

  // === Столбец "Ярлыки" === (ЗАГЛУШКА)
  labels: string[];              // [] — пустой массив на первом этапе
  activePromo: string | null;    // null
  promoExpired: boolean;         // false

  // === Столбец "Рейтинг" ===
  rating: number | null;         // средний рейтинг 1-5 (null если нет данных)
  reviewsCount: number;          // количество отзывов (0 если нет)

  // === Столбец "Цена" ===
  deliveryPrice: number | null;  // цена поставки (null если нет данных)
  sppPercent: number | null;     // скидка СПП в % (null = заглушка)
  buyerPrice: number | null;     // цена покупателю (null = заглушка)

  // === Столбец "Остаток" ===
  stockTotal: number;            // остаток всего, шт
  stockValue: number;            // стоимость остатков = stockTotal × deliveryPrice
  stockByWarehouse: { warehouse: string; qty: number }[];

  // === Столбец "Корзины и заказы общие" ===
  cartsTotal: number;            // корзины за период (0 = заглушка)
  ordersCount: number;           // количество заказов за период
  ordersSum: number;             // сумма заказов по ценам поставки

  // === Столбец "Затраты на рекламу" ===
  adsSpendTotal: number;         // общая сумма затрат на рекламу за период
  adsCartsCount: number;         // корзины с рекламы (0 = заглушка)
  adsOrdersCount: number;        // заказы с рекламы (0 = заглушка)

  // === Столбцы кампаний (Uni / Аукцион / CPC) ===
  campaigns: CampaignsByType;

  // === Столбец "Видимость" === (ЗАГЛУШКА)
  visibility: null;              // null на первом этапе
}

interface CampaignsByType {
  uni: CampaignInfo | null;
  auction: CampaignInfo | null;
  cpc: CampaignInfo | null;
}

interface CampaignInfo {
  campaignId: number;
  campaignName: string;
  type: number;                  // оригинальный type из WB API
  status: 'active' | 'paused' | 'archived';
  bid: number;                   // текущая ставка (Р)
  dailyBudget: number | null;    // дневной бюджет
  spend: number;                 // расход за период
}
```
