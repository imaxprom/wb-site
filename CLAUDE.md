# WB Parser — Правила работы

## Границы проекта
- Рабочая директория: `/Users/octopus/Projects/website`
- ЗАПРЕЩЕНО читать, писать, выполнять команды за пределами этой папки
- ЗАПРЕЩЕНО устанавливать глобальные пакеты (`npm install -g`)
- ВСЕ зависимости только локальные (`npm install` без `-g`)
- ВСЕ данные хранятся внутри проекта или в браузере (IndexedDB)
- НЕ обращаться к файлам других проектов в `/Users/octopus/Projects/`
- НЕ читать и не модифицировать системные файлы (`~/.zshrc`, `~/.npmrc`, `~/.claude/` и т.д.)

## Стек технологий
- Next.js 16 + TypeScript + Tailwind CSS 4
- xlsx-js-style — генерация Excel с стилями
- idb — IndexedDB для хранения данных в браузере
- recharts — графики
- SheetJS (xlsx) — парсинг Excel

## Структура проекта
```
src/
  app/           — страницы (Next.js App Router)
  components/    — React компоненты
  lib/           — бизнес-логика, утилиты, API
  types/         — TypeScript типы
public/          — статические файлы
```

## Запреты
- НЕ создавать файлы вне `/Users/octopus/Projects/website`
- НЕ читать файлы из других проектов или домашней директории
- НЕ использовать `pip install` / `python` для рабочих задач проекта
- НЕ модифицировать глобальные конфиги (`~/.claude/`, `~/.npm/`, `~/.zshrc`)
- НЕ устанавливать глобальные пакеты
- НЕ обращаться к Docker, базам данных или сервисам вне проекта

## Разрешения
- Читать файлы из Telegram tmp (`/Users/octopus/Library/Containers/ru.keepcoder.Telegram/Data/tmp/`) ТОЛЬКО по прямому запросу пользователя для анализа присланных файлов
- Выполнять `npm install` (локально) для добавления зависимостей
- Запускать `next dev` / `next build` для разработки
- Обращаться к WB API через серверные роуты проекта

## Dev сервер
